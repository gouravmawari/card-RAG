# API status — quick reference

**Server:** run from project root with:
```
./.venv/bin/uvicorn app.main:app --app-dir backend --reload --host 127.0.0.1 --port 8000
```
Or `cd backend && ../.venv/bin/uvicorn app.main:app --reload`. Visit http://127.0.0.1:8000/docs.

## Auth flow in Postman

1. `POST /api/v1/auth/register`  body: `{ "email": "...", "password": "...", "name": "..." }`  → returns `access_token`.
2. `POST /api/v1/auth/login`     body: `{ "email": "...", "password": "..." }`  → returns `access_token`.
3. All protected endpoints: add header `Authorization: Bearer <access_token>`. Token expires in ~1 hour — re-login.
4. `GET  /api/v1/auth/me`        → sanity-check the token returns your user_id.

## Ingestion

5. `POST /api/v1/ingest/pdf`     form-data: `file` (pdf), `subject`, `board`, `chapter`, `klass`. Returns `source_id`.
6. `GET  /api/v1/ingest/sources/{source_id}` → poll until `status=completed`.
7. `GET  /api/v1/ingest/sources` → list all your uploaded PDFs.

## Session lifecycle

8. `POST /api/v1/sessions/create`       body: `{ "source_id": "...", "num_cards": 1-15, "scheduled_for"?: iso, "focus_topics"?: [...] }`.
9. `POST /api/v1/sessions/{id}/start`   → triggers LLM generation (~10-30s). Returns the cards.
10. `POST /api/v1/sessions/{id}/answer` body: `{ "card_id": "...", "user_answer": "...", "used_hint": bool, "is_skipped": bool }`. Call per card.
11. `POST /api/v1/sessions/{id}/finalize` → LLM grades long-answers, returns `final_report_json`.
12. `GET  /api/v1/sessions/{id}`        → full session + cards + reviews.
13. `GET  /api/v1/sessions`             → list your sessions (optional `?status=completed`).

## History / analytics

14. `GET /api/v1/sessions/history/hinted-cards`  → every card you've ever clicked hint on.
15. `GET /api/v1/sessions/history/skipped-cards` → every card you've ever skipped.
16. `GET /api/v1/sessions/history/weak-topics`   → topics with high incorrect+skipped counts (drives retests).

## Retest flow

Create a new session for an existing source. If you don't pass `focus_topics`, the backend auto-picks
your weak topics for that source (intersected with the PDF's topics) and mixes in random ones.
